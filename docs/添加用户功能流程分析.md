# 添加用户功能流程分析

## 概述

本文档详细分析了添加用户功能的完整实现流程，从前端用户操作到后端 Mock 数据拦截的整个数据流向。

## 完整的数据流向图

```
用户操作 → 前端组件 → API调用 → HTTP请求 → Mock拦截 → 返回数据 → 更新界面
```

### 第4步：axios 拦截器处理请求

**文件：** `src/api/request.js`

```javascript
import axios from 'axios'
import { ElMessage } from 'element-plus'

// 创建 axios 实例
const service = axios.create({
  baseURL: process.env.VUE_APP_BASE_API,  // 从环境变量读取：'/api'
  timeout: 5000
})

// 请求拦截器
service.interceptors.request.use(
  (config) => {
    // 如果有 token，添加到请求头
    if (localStorage.getItem('token')) {
      config.headers.Authorization = localStorage.getItem('token')
    }

    // 重要：URL拼接逻辑
    // config.url = '/users'
    // config.baseURL = '/api'
    // 最终请求URL = baseURL + url = '/api' + '/users' = '/api/users'

    return config
  },
  (error) => {
    return Promise.reject(new Error(error))
  }
)

// 响应拦截器
service.interceptors.response.use(
  (response) => {
    const { data, meta } = response.data

    // 检查响应状态码
    if (meta.status === 200 || meta.status === 201) {
      return data  // 只返回 data 部分，不包含 meta
    } else {
      ElMessage.error(meta.msg)
      return Promise.reject(new Error(meta.msg))
    }
  },
  (error) => {
    console.log(error.response)
    error.response && ElMessage.error(error.response.data)
    return Promise.reject(new Error(error.response.data))
  }
)

export default service
```

**环境变量配置：** `.env.development`

```bash
VUE_APP_BASE_API = '/api'
```

**URL 拼接过程：**

```
代码中：url: '/users'
配置中：baseURL: '/api'
实际请求：http://localhost:8080/api/users
```

### 第5步：Mock.js 拦截请求

**文件：** `src/mock/index.js`

```javascript
import Mock from 'mockjs'

// 缓存用户数据
const cachedUsersList = [/* 30个用户数据 */]

// 通用响应格式
const successResponse = (data) => ({
  data,
  meta: {
    status: 200,
    msg: '操作成功'
  }
})

// 模拟添加用户接口
Mock.mock(/\/api\/users$/, 'post', (options) => {
  // 1. 解析请求体数据
  const body = JSON.parse(options.body)
  // body = {
  //   username: "张三",
  //   email: "zhangsan@example.com",
  //   mobile: "13800138001",
  //   role_name: "普通用户"
  // }

  // 2. 生成新的用户ID（当前最大ID + 1）
  const maxId = Math.max(...cachedUsersList.map(u => parseInt(u.id)))
  const newId = String(maxId + 1)

  // 3. 创建新用户对象
  const newUser = {
    id: newId,
    username: body.username,
    email: body.email,
    mobile: body.mobile,
    role_name: body.role_name,
    mg_state: true  // 默认启用
  }

  // 4. 添加到缓存列表的开头
  cachedUsersList.unshift(newUser)

  // 5. 返回成功响应
  return successResponse({
    ...newUser,
    message: '用户添加成功'
  })
})
```

### 第6步：Mock.match 匹配规则详解

#### 正则表达式 `/\/api\/users$/` 解释

| 字符 | 含义 | 说明 |
|------|------|------|
| `\/` | 匹配 `/` | 转义字符，匹配斜杠 |
| `api` | 匹配 `api` | 字面量字符串 |
| `\/` | 匹配 `/` | 转义字符，匹配斜杠 |
| `users` | 匹配 `users` | 字面量字符串 |
| `$` | 匹配结尾 | 确保字符串到此结束 |

#### 匹配示例

| URL | 是否匹配 | 原因 |
|-----|---------|------|
| `http://localhost:8080/api/users` | ✅ 匹配 | 完全符合 |
| `https://example.com/api/users` | ✅ 匹配 | 完全符合 |
| `/api/users` | ✅ 匹配 | 完全符合 |
| `http://localhost:8080/api/users/123` | ❌ 不匹配 | 有额外路径 |
| `http://localhost:8080/api/GOODS` | ❌ 不匹配 | 大小写不同 |
| `http://localhost:8080/api/users?page=1` | ❌ 不匹配 | 有查询参数 |

#### 多种请求方法的 Mock 匹配

```javascript
// GET 请求 - 获取用户列表
Mock.mock(/\/api\/users(\?.*)?$/, 'get', handler)

// POST 请求 - 添加用户
Mock.mock(/\/api\/users$/, 'post', handler)

// PUT 请求 - 编辑用户
Mock.mock(/\/api\/users\/\d+$/, 'put', handler)

// DELETE 请求 - 删除用户
Mock.mock(/\/api\/users\/\d+$/, 'delete', handler)
```

## 重要概念解释

### 1. axios baseURL 的作用

```javascript
const service = axios.create({
  baseURL: '/api'  // 基础URL
})

// 当调用 request({ url: '/users' }) 时
// 实际请求的URL = baseURL + url = '/api' + '/users' = '/api/users'
```

**优点：**
- 统一管理基础URL
- 代码更简洁（不需要每次写完整路径）
- 便于环境切换（开发/测试/生产）

### 2. Mock.js 的工作原理

```javascript
Mock.mock( url, method, handler )
           ↑      ↑       ↑
         匹配URL 请求方法 处理函数
```

**拦截时机：**
- Mock.js 在 axios 发送请求前拦截
- 不真正发送HTTP请求
- 直接返回模拟数据

**好处：**
- 前后端分离开发
- 不依赖后端接口
- 可以模拟各种场景

### 3. Promise 的 async/await 处理

```javascript
const handleAddSubmit = async () => {
  try {
    // 等待异步操作完成
    await addFormRef.value.validate()  // 表单验证
    await addUser(addForm.value)        // API调用

    // 成功后的操作
    ElMessage.success('用户添加成功')
  } catch (error) {
    // 错误处理
    ElMessage.error('添加失败：' + error.message)
  }
}
```

## 总结

添加用户功能的完整流程展示了前后端分离架构中的数据流转：
